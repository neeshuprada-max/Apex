# Contributing to Apex Gaming Rentals

Thank you for your interest in contributing! This guide will help you understand how to contribute.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/Apex.git`
3. Create a feature branch: `git checkout -b feature/your-feature-name`
4. Install dependencies: `npm install`
5. Start development server: `npm run dev`

## Development Guidelines

### Code Style
- Use ES6+ features
- Follow existing code structure
- Use meaningful variable names
- Add comments for complex logic

### File Organization
```
src/
├── components/      # Reusable React components
├── hooks/          # Custom React hooks
├── utils/          # Utility functions
├── styles/         # Global styles
└── App.jsx         # Main application component
```

### Commit Message Format
- **Add:** for new features
- **Fix:** for bug fixes
- **Update:** for modifications
- **Remove:** for deletions
- **Refactor:** for code restructuring
- **Docs:** for documentation updates

Example: `git commit -m "Add: payment integration feature"`

## Pull Request Guidelines

1. Update your branch with latest changes
2. Test your changes thoroughly
3. Write clear commit messages
4. Submit PR with detailed description
5. Address review feedback

## Bug Reports

Include:
1. Clear description of the bug
2. Steps to reproduce
3. Expected vs actual behavior
4. Screenshots if applicable
5. Your environment (browser, OS, device)

## Feature Requests

Include:
1. Clear description of the feature
2. Why it's needed
3. Proposed solution
4. Alternative approaches
